1 Install System Dependencies

# Install Homebrew (skip if already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Python 3.12
brew install python@3.12

# PostgreSQL 16
brew install postgresql@16

# Start PostgreSQL now and on every login
brew services start postgresql@16

# pgvector extension
brew install pgvector
# Python
sudo apt update
sudo apt install python3.12 python3.12-venv python3.12-dev -y

# PostgreSQL 16
sudo apt install postgresql-16 postgresql-client-16 -y
sudo systemctl start postgresql
sudo systemctl enable postgresql

# pgvector
sudo apt install postgresql-16-pgvector -y
Fix for: ConnectionTimeout: connection timeout expired on port 5432 PostgreSQL is not installed or not running on this machine. Use Option A (Docker) — it takes 3 minutes and requires no manual extension setup.

Option A — Docker Desktop (recommended for Windows)

Install Docker Desktop for Windows, then run in PowerShell:

# Pull and start postgres + pgvector in one command
docker run -d `
  --name cc-rewards-db `
  -e POSTGRES_DB=credit_rewards `
  -e POSTGRES_USER=postgres `
  -e POSTGRES_PASSWORD=postgres `
  -p 5432:5432 `
  pgvector/pgvector:pg16

# Verify container is running
docker ps

Set these in your .env file:

PGHOST=localhost
PGPORT=5432
PGDATABASE=credit_rewards
PGUSER=postgres
PGPASSWORD=postgres

Option B — Native PostgreSQL installer

Download the PostgreSQL 16 installer from postgresql.org/download/windows. After install, open Stack Builder → select PostgreSQL 16 → tick pgvector → Install.

2 Get the Code

git clone <your-repo-url>
cd capstone-intelligent-credit_card-optimization_agent

3 Create the Python Virtual Environment

# Create a fresh venv from the project root
python3 -m venv .venv

# Activate — macOS / Linux / WSL2
source .venv/bin/activate

# Your prompt now shows (.venv) — you're inside the venv
Moved the project folder? Virtual environments store absolute paths. If you moved or copied the project directory, the venv breaks. Delete it and recreate:

rm -rf .venv && python3 -m venv .venv && source .venv/bin/activate

4 Install Python Packages

# Upgrade pip first
pip install --upgrade pip

# Install all dependencies
pip install -r requirements.txt
Corporate network / TLS-inspecting proxy? If pip fails with SSL: CERTIFICATE_VERIFY_FAILED, use pre-built wheels to skip the Rust compile step:

pip install --only-binary=:all: -r requirements.txt

5 Set Up the PostgreSQL Database

5a — Create the Database

# On Homebrew, your OS user is already a superuser — no password needed
createdb credit_rewards
# Switch to the postgres system user first
sudo -u postgres createdb credit_rewards
sudo -u postgres psql -c "CREATE USER $(whoami) WITH SUPERUSER;"

5b — Enable pgvector Extension

psql -d credit_rewards -c "CREATE EXTENSION IF NOT EXISTS vector;"

5c — Verify

psql -d credit_rewards -c "SELECT extversion FROM pg_extension WHERE extname='vector';"
# Expected output:
#  extversion
# ------------
#   0.8.x

6 Configure the .env File

cp .env.example .env

Open .env in any editor. You must fill in two things: your LLM provider key and your PostgreSQL username.

# ── LLM Provider — pick ONE block, comment out the others ────────────────────

# Option A: Gemini (free tier — easiest to start)
LLM_MODEL=gemini/gemini-2.5-flash
GEMINI_API_KEY=your_gemini_key_here

# Option B: Anthropic / Claude
# LLM_MODEL=anthropic/claude-opus-4-8
# ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxx

# Option C: OpenAI
# LLM_MODEL=openai/gpt-4o-mini
# OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxx

# Option D: Local Ollama (no key, fully offline)
# LLM_MODEL=ollama/llama3.1
# OLLAMA_API_BASE=http://localhost:11434

# ── PostgreSQL ────────────────────────────────────────────────────────────────
PGHOST=localhost
PGPORT=5432
PGDATABASE=credit_rewards
PGUSER=your_os_username     # run: whoami
PGPASSWORD=                   # leave blank for Homebrew / socket auth

# ── Embeddings (no API key needed) ────────────────────────────────────────────
# auto = tries fastembed, falls back to hashing
# hashing = no model download (best on corporate/restricted networks)
# fastembed = downloads ONNX model on first run (~90 MB)
EMBED_BACKEND=auto
EMBED_MODEL=BAAI/bge-small-en-v1.5
EMBED_DIM=384
Find your PostgreSQL username Run whoami in the terminal — that is your PGUSER value on a Homebrew setup.

7 Run the Seed Pipeline

This one command: creates the database schema → ingests all 5 card documents → populates reward rules.

# Make sure venv is activated
python3 seed.py

Expected output:

[1/3] Initialising schema ...  done
[2/3] Ingesting card documents ...
  axis_atlas.md             → 12 chunks
  hdfc_diners_club_black.md → 14 chunks
  hdfc_infinia.md           → 11 chunks
  amex_platinum_travel.md   → 13 chunks
  sbi_cashback.md           →  9 chunks
[3/3] Seeding structured reward rules ...  55 rules, 18 transfer partners
Setup complete.

Verify the Data

psql -d credit_rewards -c "SELECT COUNT(*) FROM card_documents;"   # → 5
psql -d credit_rewards -c "SELECT COUNT(*) FROM reward_rules;"       # → 55
psql -d credit_rewards -c "SELECT COUNT(*) FROM document_chunks;"    # → ~59
LLM-powered rule extraction (optional) The default seed uses hand-authored fallback rules. To re-extract rules using the LLM from the ingested documents (takes ~2–3 minutes, needs API key):

python3 seed.py --extract

Troubleshooting

⚠ ConnectionTimeout: connection timeout expired (Windows / port 5432)
PostgreSQL is not installed on this machine. Quickest fix — use Docker: 
docker run -d --name cc-rewards-db `
  -e POSTGRES_DB=credit_rewards `
  -e POSTGRES_USER=postgres `
  -e POSTGRES_PASSWORD=postgres `
  -p 5432:5432 pgvector/pgvector:pg16
Then set PGUSER=postgres and PGPASSWORD=postgres in .env.

No Docker? Use the native installer: postgresql.org/download/windows → run installer → open Stack Builder → install pgvector.
⚠ psql: error: connection refused
PostgreSQL is not running. Start it: 
# macOS
brew services start postgresql@16
# Linux
sudo systemctl start postgresql
⚠ extension "vector" does not exist
pgvector is not installed or not enabled in this database. 
# macOS — install the package
brew install pgvector
# Then enable it in the database
psql -d credit_rewards -c "CREATE EXTENSION vector;"
⚠ LLM provider 'gemini' is not configured: GEMINI_API_KEY is not set
Open .env and add your key: 
GEMINI_API_KEY=your_key_here
⚠ streamlit: bad interpreter — or — uvicorn: command not found
The venv was moved or is corrupted. Recreate it: 
rm -rf .venv
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
⚠ SSL: CERTIFICATE_VERIFY_FAILED (corporate proxy)
The project uses truststore to inject the OS certificate store automatically. If the error persists: 
pip install truststore
# Or set the CA bundle path in .env
REQUESTS_CA_BUNDLE=/path/to/corporate-ca.pem
⚠ pip fails to build "tokenizers" from source
Use pre-built binary wheels to skip the Rust compilation: 
pip install --only-binary=:all: -r requirements.txt
⚠ seed.py runs but reward_rules count is 0
Run the LLM-powered extraction step (requires an API key): 
python3 seed.py --extract

Sharing One PostgreSQL with the Team

Instead of every person installing PostgreSQL, one machine hosts the DB and teammates point their .env at it.

When to use this Best for demos and code reviews where the team doesn't want to each seed their own database. One person (the host) seeds the data once; everyone else just sets PGHOST to the host's IP.

Step A — Allow remote connections on the host machine (macOS)

# Find the config file location
psql -c "SHOW config_file;"
# Usually: /opt/homebrew/var/postgresql@16/postgresql.conf

# 1. Edit postgresql.conf
#    Find:   listen_addresses = 'localhost'
#    Change: listen_addresses = '*'
nano /opt/homebrew/var/postgresql@16/postgresql.conf

# 2. Edit pg_hba.conf — allow password auth from any IP
#    Add at the bottom:
#    host  credit_rewards  all  0.0.0.0/0  scram-sha-256
nano /opt/homebrew/var/postgresql@16/pg_hba.conf

# 3. Set a password for the DB user
psql -c "ALTER USER shashank.rai PASSWORD 'choose_a_password';"

# 4. Restart PostgreSQL to apply changes
brew services restart postgresql@16

# 5. Find your machine's local IP — share this with teammates
ipconfig getifaddr en0
# Example: 192.168.1.42

Step B — Test connectivity from a teammate's Windows machine

# In PowerShell — should say TcpTestSucceeded : True
Test-NetConnection -ComputerName 192.168.1.42 -Port 5432

Step C — Teammates set .env to point at the host

# Each teammate's .env — replace IP and password
PGHOST=192.168.1.42
PGPORT=5432
PGDATABASE=credit_rewards
PGUSER=shashank.rai
PGPASSWORD=choose_a_password
Teammates do NOT run seed.py Data is already on the host machine. They only need: Python + venv + pip install -r requirements.txt + the .env above + run the app.

Option: Supabase free cloud PostgreSQL (accessible from anywhere)

Supabase has pgvector built in — good for demos accessible outside a local network.

StepAction
1Create a free project at supabase.com
2Dashboard → Settings → Database → copy connection details
3Set PGHOST=db.<ref>.supabase.co, PGUSER=postgres, PGPASSWORD=<your password> in .env
4Run python3 seed.py once to create schema and seed data
5Share the same .env values with teammates — everyone connects to the same cloud DB
